the data & API (we give you this)

One endpoint, everything in it

You don't need to find or build an API. Call this URL — it returns all the data, nested:

https://raw.githubusercontent.com/android-assesment/notes/refs/heads/main/data.json

The response is shaped like this:

{
  "categories": [
    {
      "name": "Android Development",
      "courses": [
        {
          "title": "Kotlin Fundamentals", "rating": 4.7, "durationHours": 6.5,
          "thumbnailUrl": "...", "level": "Beginner",
          "instructor": { "name": "Aarav Sharma", "avatarUrl": "...", "title": "..." },
          "lessons": [
            { "title": "Welcome & Setup", "durationMinutes": 8, "isFree": true, "content": "..." }
          ]
        }
      ]
    }
  ]
}

So: categories → courses → lessons. Home reads categories + courses; Course Detail reads one course + its lessons; Lesson reads one lesson.

build it — step by step

Just follow these

  1. Create a new Android project in Kotlin (Android Studio).
  2. Add Retrofit + coroutines and fetch the JSON from the API URL above.
  3. Screen 1 — Home: show the categories row and the courses list (thumbnail, level, title, instructor, rating).
  4. Screen 2 — Course Detail: tap a course → hero image, info, instructor card, and its lessons (mark free vs locked).
  5. Screen 3 — Lesson: tap a lesson → the video-player screen (player header + lesson list).
  6. Load all images from their URLs with Coil or Glide.
  7. Show a loading state and an error message for the network call.
  8. Write at least one unit test.
  9. Use AI tools (Claude, Cursor, Copilot, Gemini) throughout — that's expected here.
  10. Add a README (below) and push everything to a public GitHub repo.
