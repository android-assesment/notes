the brief

You'll build a tiny Android app that talks to a real API. Treat it like a small slice of production work — clean, readable, and tested where it matters.

Use AI tools freely; that's exactly how you'll work here. Don't over-build — a small, solid app beats a big, messy one.

your task

Build & ship a mini app

  • Build a small Android app in Kotlin.
  • Fetch data from any free public API that needs no auth — e.g. JSONPlaceholder, a public weather or crypto API. Your choice.
  • Show the results in a scrollable list.
  • Tapping an item opens a detail screen for that item.
  • Handle loading and error states — don't assume the happy path.
  • Use a sensible architecture (MVVM or similar) with Retrofit + coroutines for networking.
  • Include at least one unit test that actually tests something.
the part that matters most
AI Workflow · README

Along with your code, add a short README that shows how you used AI:

  • Which AI tools you used (Claude, Cursor, GitHub Copilot, Gemini in Android Studio, etc.).
  • 2–3 of the actual prompts you gave them.
  • One place AI saved you time — and one place its output was wrong or you had to override it.

We're not testing whether you used AI. We're testing whether you use it with judgment.

what we'll look at

How we evaluate

  • Correct handling of async work, networking and errors.
  • Thoughtful use of AI — not blind copy-paste.
  • A meaningful test, not just a placeholder.
